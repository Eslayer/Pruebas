FLUJO DE DATOS
==============

1. FLUJO DE PEDIDO
   Cliente → Frontend → API Gateway → Eureka Server
   → Auth Service (autenticación)
   → Menu Service (obtener platos)
   → Inventory Service (validar stock)
   → Orders Service (crear pedido)
   → Kitchen Service (notificar cocina)
   → Payment Service (procesar pago)
   → Notification Service (confirmar pedido)

2. FLUJO DE PAGO
   Cliente → Frontend → API Gateway → Eureka Server
   → Payment Service
   → Orders Service (actualizar estado)
   → Notification Service (confirmar pago)

3. FLUJO DE COCINA
   Kitchen Service → Menu Service (consumir ingredientes)
   → Inventory Service (actualizar stock)
   → Notification Service (alertas de stock bajo)

CONCLUSIÓN
==========
La arquitectura de microservicios de QuickBite sigue los principios de diseño de sistemas distribuidos modernos, con separación clara de responsabilidades, comunicación asíncrona donde es apropiado, y patrones de resiliencia para garantizar alta disponibilidad y escalabilidad.
